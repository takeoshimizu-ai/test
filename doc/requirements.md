# 機能要件

## 概要
本サービスは、ブラウザやHTTPクライアントからGETリクエストで呼び出せる2つのAPIを提供します。

## エンドポイント
- `/api/mul` — 掛け算API
- `/api/div` — 割り算API

## HTTPメソッド
- `GET`

## リクエスト
- クエリ文字列パラメータとして次を必須とする。
  - `A`：数値（整数または小数）
  - `B`：数値（整数または小数）
- 例: `/api/mul?A=1.5&B=2`

## 入力バリデーション
- `A`/`B` が存在しない、または数値に変換できない場合は HTTP 400 を返す（本文は短いプレーンテキストのエラーメッセージ）。

## ビジネスロジック
- `/api/mul`：戻り値 = A × B
- `/api/div`：戻り値 = A ÷ B
- 丸めルール：結果は小数点以下最大6桁で四捨五入し、末尾の不要なゼロは削除する（例: `2.500000` → `2.5`, `3.000000` → `3`）。

## 特殊エラー
- `/api/div` において `B == 0` の場合、HTTP 401 を返す（本文は短いプレーンテキストのエラーメッセージ）。

## レスポンス
- レスポンス形式：プレーンテキスト（数値のみ、改行なし）
- 成功時の例:
  - リクエスト: `/api/mul?A=1.5&B=2`
  - レスポンス(200): `3`
  - リクエスト: `/api/div?A=1&B=3`
  - レスポンス(200): `0.333333`  (小数点以下6桁に丸め)

## エラーレスポンス例
- パラメータ欠落/非数値: HTTP 400, 本文例: `Invalid or missing parameter A or B`
- ゼロ除算: HTTP 401, 本文例: `Division by zero`

## 注記
- ブラウザでURLを入力して応答が画面に表示されれば要件を満たす（簡易なブラウザ表示／APIとしての利用を両方想定）。
