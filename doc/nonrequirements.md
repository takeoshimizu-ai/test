# 非機能要件

## ランタイム / プラットフォーム
- Azure Functions（Consumption Plan）
- リージョン: East Japan（東日本）
- 開発言語: Python 3.11

## 認証 / セキュリティ
- 認証は不要（パブリックに公開される想定）。
- 公開APIであるため、運用時は必要に応じてIP制限やWAFなどを検討すること。

## CORS
- すべてのオリジンを許可する（`Access-Control-Allow-Origin: *`）。

## レスポンス形式
- プレーンテキストを返す（JSONではない）。

## エラーハンドリング
- 入力エラー: HTTP 400
- ゼロ除算: HTTP 401
- エラーメッセージは短いプレーンテキスト

## ログ・監視
- Application Insights 等で基本的なリクエスト/エラーログを収集することを推奨。

## デプロイ / CI
- GitHub Actions や Azure CLI を利用した自動デプロイを想定。

## スケーリング / 可用性
- Consumption Plan により要求に応じ自動スケールする。
- 高可用性や厳格なSLAが必要な場合はプラン変更や複数リージョン展開を検討すること。

## 開発者向け（ローカル実行）
- Azure Functions Core Tools を使ってローカルで動作検証できる。
- Python 3.11 の実行環境を用意すること。

## ストレージ / 設定
- Azure Functions の実行に必要なストレージアカウントが必要（デプロイ時に作成/指定）。
- 機密値は Key Vault またはアプリ設定で管理すること。

## その他
- 追加要件が無いことを確認済み。将来的に認証・モニタリング要件が追加される場合はドキュメントを更新してください。
